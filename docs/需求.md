# A股量化交易策略工具 — 产品需求文档

**版本**: v0.1
**日期**: 2026-03-17
**状态**: 草稿

---

## 一、项目背景与目标

本工具旨在帮助个人投资者借鉴头部量化基金的选股逻辑，通过系统化的数据分析和策略验证，构建适合个人操作的A股量化投资体系。核心理念：**站在巨人的肩膀上，把机构的选股逻辑平民化**。

---

## 二、核心功能模块

### 模块 1：量化基金筛选器

**目标**：从市场上筛选出规模大、持续盈利能力强的量化私募/公募基金。

**筛选维度**：

| 维度 | 指标 | 说明 |
|------|------|------|
| 规模 | AUM ≥ 50亿 | 过滤小基金，确保策略有足够资产验证 |
| 历史收益 | 近3年年化收益 ≥ 15% | 剔除运气因素，要求持续性 |
| 回撤控制 | 最大回撤 ≤ 20% | 风险调整后的收益能力 |
| 夏普比率 | ≥ 1.5 | 综合风险收益比 |
| 存续年限 | ≥ 3年 | 经历完整牛熊周期 |
| 超额收益 | 相对沪深300年化超额 ≥ 10% | 剔除纯Beta收益 |

**数据来源**：
- 私募排排网 / 朝阳永续 / 格上财富（私募数据）
- Wind / Choice（公募量化基金数据）
- 基金业协会备案信息

**产出**：目标基金池（预计 20~50 只）

---

### 模块 2：基金策略分析

**目标**：通过持仓披露、公开报告、因子归因，反推基金的核心选股逻辑。

**分析方法**：

#### 2.1 持仓分析
- 解析季报/年报中的前十大重仓股
- 统计行业分布、市值偏好（大/中/小盘）
- 持仓换手率估算（通过相邻季报对比）

#### 2.2 因子归因
基于 Barra 多因子模型，分解基金收益来源：
- **风格因子**：价值(Value)、动量(Momentum)、质量(Quality)、低波动(LowVol)、规模(Size)
- **行业因子**：识别基金的行业偏好
- **Alpha因子**：无法被已知因子解释的超额收益（核心竞争力）

#### 2.3 策略类型识别

| 策略类型 | 特征 | 代表基金 |
|----------|------|----------|
| 指数增强 | 跟踪指数 + 少量偏离 | 幻方、九坤、明汯 |
| 中性策略 | 市场中性，对冲Beta | 部分私募 |
| CTA趋势 | 期货/商品趋势跟踪 | 盘京、进化论 |
| 多因子选股 | 纯股票多头，因子驱动 | 宽德、衍复 |

#### 2.4 公开信息挖掘
- 基金经理访谈、路演纪要
- 公司研究报告
- 监管披露的持仓数据（如龙虎榜、大宗交易）

### 模块 3：个人选股方案生成

**目标**：将机构策略的核心因子，转化为个人投资者可执行的选股规则。

**核心设计原则**：
- 规则透明、可解释（非黑箱）
- 适合个人仓位规模（无需做空、无需期货）
- 调仓频率合理（月度/季度，降低摩擦成本）

#### 3.1 因子库构建

**基本面因子**：
- ROE（净资产收益率）> 15%，且连续3年稳定
- 净利润增速 > 20%
- 毛利率趋势向上
- 经营现金流/净利润 > 0.8（确认利润质量）

**估值因子**：
- PE/PEG 相对历史分位 < 50%
- PB 低于行业中位数
- EV/EBITDA 合理区间

**技术/动量因子**：
- 过去 6 个月相对收益 > 0（动量效应）
- 成交量异常放大信号
- 布林带/均线突破

**资金流向因子**：
- 北向资金持续净买入
- 机构调研频次增加
- 融资余额变化趋势

#### 3.2 综合评分模型

```
综合得分 =
  基本面得分 × 40% +
  估值得分   × 25% +
  动量得分   × 20% +
  资金得分   × 15%
```

#### 3.3 股票池筛选流程

```
全市场 5000+ 股票
    ↓ 流动性过滤（日均成交 > 1亿）
    ↓ 排除 ST/*ST
    ↓ 排除上市不足1年
    ↓ 基本面因子初筛
    ↓ 综合评分排名
    ↓ 人工确认（行业分散度）
    → 目标持仓组合（20~30只）
```

### 模块 4：策略历史回测验证

**目标**：用历史数据验证选股策略的有效性，评估风险收益特征。

#### 4.1 回测框架设计

**基本参数**：
- 回测区间：2015-01-01 至今（覆盖牛熊转换）
- 调仓频率：月度 / 季度（可配置）
- 基准：沪深300 / 中证500 / 中证1000
- 初始资金：100万（模拟个人账户）
- 手续费：万分之三（买卖双向）
- 滑点：0.1%

**分仓方式**：
- 等权重（每只股票相同仓位）
- 评分加权（按综合评分分配权重）
- 风险平价（按波动率倒数分配）

#### 4.2 回测评估指标

| 指标 | 说明 | 目标值 |
|------|------|--------|
| 年化收益率 | 复合年化 | ≥ 20% |
| 最大回撤 | 净值曲线最大跌幅 | ≤ 25% |
| 夏普比率 | 超额收益/波动率 | ≥ 1.5 |
| 卡玛比率 | 年化收益/最大回撤 | ≥ 1.0 |
| 信息比率 | 超额收益/跟踪误差 | ≥ 1.0 |
| 胜率 | 上涨股票占比 | ≥ 55% |
| 年换手率 | 买卖频率 | ≤ 300% |

#### 4.3 分阶段验证

| 阶段 | 时间 | 意义 |
|------|------|------|
| 牛市 | 2019-2021 | 策略在上涨中的弹性 |
| 熊市 | 2021-2022 | 策略的回撤控制能力 |
| 震荡市 | 2016-2018 | 策略在横盘中的表现 |
| 全周期 | 2015至今 | 综合验证 |

#### 4.4 因子有效性检验

- **IC 分析**：因子值与下期收益的相关系数（IC 均值 > 0.05 为有效）
- **IC_IR**：IC 均值/IC 标准差（> 0.5 为稳定）
- **分层回测**：按因子分 5 档，验证单调性
- **因子衰减**：验证因子在不同持仓周期的有效性

## 三、技术架构建议

### 3.1 数据层

```
数据源
├── Tushare Pro / AKShare（免费，适合起步）
├── Wind API（专业，付费）
└── Choice 金融终端（付费）

数据库
├── PostgreSQL（结构化行情、财务数据）
├── Redis（实时行情缓存）
└── MinIO / 本地文件（历史数据归档）
```

### 3.2 计算层

```
Python 生态
├── pandas / numpy（数据处理）
├── backtrader / bt / zipline（回测框架）
├── alphalens（因子分析）
├── pyfolio（组合绩效分析）
└── scikit-learn（机器学习因子挖掘，进阶）
```

```
可视化方案
├── Streamlit（快速搭建Web界面，推荐起步）
├── Dash / Plotly（交互式图表）

```

## 四、数据获取方案

### 4.1 行情数据
- **日线数据**：Tushare `daily`、AKShare `stock_zh_a_hist`
- **分钟数据**：Tushare Pro（需积分）
- **指数数据**：沪深300、中证500、中证1000

### 4.2 财务数据
- 利润表、资产负债表、现金流量表（季报/年报）
- Tushare `income` / `balancesheet` / `cashflow`

### 4.3 基金数据
- 基金持仓：Tushare `fund_portfolio`
- 基金净值：Tushare `fund_nav`
- 私募数据：需通过爬虫或第三方平台

### 4.4 替代数据（进阶）
- 舆情数据：新闻情绪评分
- 卫星/地图数据：消费/工业活动代理指标
- 招聘数据：企业扩张信号

## 五、风险与局限性说明

| 风险类型 | 说明 | 应对措施 |
|----------|------|----------|
| 过拟合风险 | 回测表现好但实盘差 | 样本外验证、Walk-Forward测试 |
| 数据挖掘偏差 | 因子在历史数据中有效，未来未必 | 保持因子逻辑可解释性 |
| 流动性风险 | 个人账户实际成交与回测有偏差 | 设置流动性门槛，保守估计滑点 |
| 策略失效风险 | 因子被大量资金拥挤后失效 | 定期重新验证因子 IC |
| 监管风险 | A股政策变化影响策略有效性 | 关注政策动向，设置人工干预入口 |

## 七、补充建议

### 7.1 关于基金持仓的局限性
公募基金持仓披露有滞后（季报滞后约15天），私募基金无强制披露义务，因此基金分析更多是方向性参考，不能直接照搬。建议结合龙虎榜、机构席位等高频信号补充。

### 7.2 关于个人账户的现实约束
- A股 T+1 交易制度，策略设计需考虑
- 印花税 0.1%（卖出单边），手续费万分之三上下
- 个人账户无法做空，需设计纯多头策略

### 7.3 关于策略可持续性
量化因子存在"因子拥挤"问题——当太多人使用同一因子时，超额收益会被摊薄。建议：
- 定期（每半年）重新跑因子 IC 测试
- 引入低相关度的多个因子，降低单一因子失效风险
- 保留人工判断窗口，不做全自动化决策

### 7.4 合规性提示
本工具仅供个人投资研究使用，不构成投资建议。数据使用需遵守相关数据服务商的使用协议。
